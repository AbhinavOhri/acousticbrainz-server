Client downloads

In order to contribute to AcousticBrainz, you will need to have a music collection that contains MBIDs. This usually means that your collection will need to have been tagged with a MusicBrainz aware tagger like Picard.

If you have a collection of music files with MBIDs, please download one of the tools below to help us bootstrap this project. This project is still in its infancy, so many things are not nearly as polished as we would like them to be. This means that you'll like need a little more technical experience in order to submit files to AcousticBrainz. We will work to improve this with full blown applications for Linux, Mac and Windows -- to be released soon.

The easiest way to contibute right now is to download the following virtual machine. Alternatively, you can download and build the source code on your own machine -- beware if you're not fully comfortable with git and compiling software you should use the VM or wait for some of our future GUI based software releases.

Experimental Virtual Machine

If you'd like to download our virtual machine that contains a minimal Linux setup and automatically uploads track results to AcousticBrainz, start by downloading the virtual machine:

  • Direct download: acousticbrainz-submitter-vm-20141018.ova (1.0GB) or
  • BitTorrent: acousticbrainz-submitter-vm-20141018.ova.torrent (21Kb)

The md5 sum for the ova file is: a9c4576a23f42c2bbc55eb01f96c6ea9

To run this virtual machine, you will need to download and install VirtualBox. Once you have finished installing VirtualBox, you can open the downloaded ova file. IMPORTANT: Do not start the virtual machine yet!

Before you start the virtual machine, select the virtual machine and click on Settings for that virtual machine. In the Settings dialog, look for the Shared Folders section. In that section, look for a folder icon with a plus on it -- click that to add a new shared folder. Select your music collection folder, and the check the Auto-mount and Read-only checkboxes (important!). Then close the settings dialog.

Now you start the virtual machine by clicking the start button. If everything went well, the system should be analyzing your collection and submitting the track information to AcousticBrainz. Sadly, we don't have much feedback about the submittion progress yet -- we'll work on that next. (If you're comfortable with Linux, you can log in with user ab, password ab and then: 

tail -f submit.log
to see the progress messages.

Installing from source

If you prefer to build your own install of this software, you'll need a recent Linux install. Follow the instructions in acousticbrainz-client.
