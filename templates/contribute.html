Contributing to AcousticBrainz

In order to contribute to AcousticBrainz, you will need to have a music collection that contains MBIDs. This usually means that your collection will need to have been tagged with a MusicBrainz aware tagger like Picard. If you have a collection of lossless FLAC files that would be ideal, but having lossly MP3s/AAC/ogg files also helps the project.

If you have a collection of music files with MBIDs, please one of the submission tools to help us bootstrap this project. This project is still in its infancy, so many things are not nearly as polished as we would like them to be. This means that you'll like need a little more technical experience in order to submit files to AcousticBrainz. We will work to improve this with full blown applications for Linux, Mac and Windows -- to be released soon.

The easiest way to contibute right now is to download the virtual machine. Alternatively, you can download and build the source code on your own machine -- beware if you're not fully comfortable with git and compiling software, you should use the VM or wait for some of our future GUI based software releases.

Experimental Virtual Machine

To submit your files with our virtual machine, you will need to download and install VirtualBox. You'll also need to download our Virtual Machine image -- the download is 1Gb large, so get started now!

Once you have finished installing VirtualBox, you can open the downloaded ova file. IMPORTANT: Do not start the virtual machine yet!

Before you start the virtual machine, select the virtual machine and click on Settings for that virtual machine. In the Settings dialog, look for the Shared Folders section. In that section, look for a folder icon with a plus on it -- click that to add a new shared folder. Select your music collection folder, and the check the Auto-mount and Read-only checkboxes (important!). Then close the settings dialog.

Now you start the virtual machine by clicking the start button. If everything went well, the system should be analyzing your collection and submitting the track information to AcousticBrainz. Sadly, we don't have much feedback about the submittion progress yet -- we'll work on that next. (If you're comfortable with Linux, you can log in with user ab, password ab and then: 

tail -f submit.log
to see the progress messages.
